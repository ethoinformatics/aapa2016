What's Currently Available

  • Noldus Observer
  • HanDBase
  • Mangold Obansys
  • prime8
  • Various database + data form apps for structured note taking

Limitations?

  • Many are proprietary
  • Limited data streams
  • No emphasis on using standard ontologies
  • Not automatically geotagged

Goal - An APP for Handhelds That Is...

  • Modular
  • Open source
  • Configurable
  • Allows multiple data streams
  • Runs on multiple platforms
  • Schemaless and supports existing schemas
  • EthoCore compliant
  • Automatic geotagging and time stamping
  • Can run on cheaper and older devices
  • Does not require continuous internet coverage

General Idea

  • App generates documents that are made up of EthoCore terms
  • Each document is a resource with a URI (uniform resource identifiers)
  • Documents can be decomposed to triple-stores and used as linked data
  • Documents can link to one or more other documents via standard predicates, each of which has a URI (taken within, belongs to, child of, associates with)
  • Media (pictures, audio recordings) can be linked by a predicate to one or more documents
  • Every document has key metadata
    • Who made it
    • When it was created
    • Time (point or series)
    • Geolocation (point or series)
  • History of revisions of documents is preserved

Some Design Decisions

  • Use HMTL, JavaScript, and CSS to create a mobile app that can be compiled and deployed on a variety of devices
  • Use JSON documents to configure aspects of the application
  • Use local storage and PouchDB on the mobile device to hold data
  • Data in PouchDB stored as JSON documents
  • Sync from PouchDB to CouchDB on a local server and from CouchDB on local server to remote server
  • Use data from operating system's location services to tag all documents with geolocation
  • (from internal GPS, cell towers, external GPS)
  • Geolocation data is updated recorded continuously

User Provides...

  • A set of controlled vocabularies
  • Design for construction of documents
  • Information about on how documents can be be linked

App Writes Data That...

  • Conforms to EthoCore and other community-designed data standards (e.g., Darwin Core)
  • Stores data as documents (linked? nested?)
  • Stores geolocation tag and time stamp tag for all documents
